# 生成结果自检 Prompt

这是一份校验层 Prompt，用于检查各层内容有没有互相越界。

用途：

- 用于在生成 `persona.md`、`lore.md`、`relationship.md`、`custom.md` 后做自检
- 目标是检查分层是否被破坏，而不是重写内容

## 自检目标

确认以下原则是否成立：

1. 网页固定数据层只包含网页有依据的角色本体和设定事实
2. 关系身份层只影响角色如何面向用户互动
3. 角色丰富层只增强表达，不改写设定
4. 各层之间没有越界污染

## 检查顺序

### 1. 检查 `persona.md`

确认：

- 是否只在描述角色本体
- 是否混入了朋友、恋人、同事等关系身份设定
- 是否把页面没有依据的内容写成了角色稳定特征
- 是否把页面原话误当成了日常对话里应直接复述的固定句子

如果有问题：

- 标记为“固定数据层越界”或“无依据补完”

### 2. 检查 `lore.md`

确认：

- 是否只保留可问答事实
- 是否混入了人格判断、关系身份判断、表现层补充
- 是否把推断写成了事实

如果有问题：

- 标记为“Lore 混入 Persona / 增强层”或“推断写成事实”

### 3. 检查 `relationship.md`

确认：

- 是否只在描述关系身份下的互动方式
- 是否改写了角色本体
- 是否发明了共同经历、共同回忆、共同设定
- 是否出现明显违背角色气质的亲密表达
- 如果参考了 `relationship_scene_builder.md`，是否把场景增强误写成了新的角色设定

如果有问题：

- 标记为“关系身份层越界”或 “关系表达 OOC”

### 4. 检查 `custom.md`

确认：

- 是否只在补高频场景、情绪层次、语言纹理、长期陪伴感、降级策略
- 是否写进了新的设定事实
- 是否把关系身份内容写成了角色本体
- 是否存在过度模板化、过度油腻、过度现代化表达
- 是否把某个单独场景的临时表达误写成了所有场景都适用的硬规则
- 是否把生气、失望、无奈、心软等不同情绪写成了同一种表达节奏
- 是否出现多个场景结构过度相似，只是换了情绪标签
- 是否把“表达方向”写成了“固定答案”
- 是否把剧情原句或角色经典台词生硬塞进日常对话，导致回复突兀

如果有问题：

- 标记为“Custom 混入设定层”或“表达出戏”

## 输出格式

如果没有问题：

- 输出：`当前分层正常，未发现明显越界。`

如果有问题：

- 按问题列出：
  - 问题层
  - 问题类型
  - 简短原因
  - 建议改回哪个文件层

## 核心原则

- 不要因为想让角色更饱满，就把增强层写进固定数据层
- 不要因为想让关系更有感觉，就改写网页提炼出的角色本体
- 不要因为想让回答更自然，就把无依据内容写成设定事实
- Prompt 用来定义生成方向，不用来写固定答案
