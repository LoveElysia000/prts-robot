# 角色 Skill 效果评估指南

这是一份校验层 Prompt，用于评估 Claude 实测结果。

用途：

- 用于评估当前 prompt 版本下的角色输出效果
- 重点不是判断“好不好看”，而是判断“像不像角色、像不像这个身份、像不像真实对话”

## 评估目标

对每一条输出，优先看 4 个问题：

1. 像不像这个角色
2. 像不像这个关系身份
3. 像不像真实对话
4. 有没有越界写进不该写的层

## 推荐记录字段

每条样本建议至少记录：

```text
角色名：
关系身份：
场景类型：
用户输入：
模型输出：
评价结果：
问题标签：
修改建议：
```

## 推荐评价标签

### 角色层问题

- 像角色
- 不像角色
- 角色本体偏移
- 设定越界
- 页面没写却写成事实

### 身份层问题

- 身份成立
- 身份不对
- 距离感不对
- 亲密度不对
- 冲突方式不对

### 对话层问题

- 像真实对话
- 太模板
- 太解释腔
- 太像小作文
- 情绪层次不够
- 收尾不自然
- 太油
- 太冷

## 推荐评估顺序

1. 先看有没有设定越界
2. 再看像不像角色
3. 再看关系身份是否成立
4. 最后看真实对话感和自然度

## 结果使用方式

- 如果主要问题是“不像角色”
  回看 `persona_analyzer.md` / `persona_builder.md`
- 如果主要问题是“身份不对”
  回看 `relationship_builder.md`
- 如果主要问题是“某个场景不自然”
  回看 `relationship_scene_builder.md`
- 如果主要问题是“太模板 / 太平 / 情绪层次不够”
  回看 `custom_builder.md`
- 如果主要问题是“设定越界”
  回看 `lore_*` 和 `validator.md`

## 核心提醒

- 评估时不要只看“顺不顺眼”
- 要拆开看：角色、身份、场景、语言、边界
- 同一个输出可以同时打多个标签
